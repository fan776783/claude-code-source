# 07. 接口与协议

## 1. 核心内部契约

### 1.1 Command

定义文件：

- `src/types/command.ts`

作用：

- 统一 slash command、本地命令、本地 JSX 命令
- 把命令式交互与模型回合接起来

当前高频类型：

- `PromptCommand`
- `LocalCommand`
- `LocalJSXCommand`
- `Command`

### 1.2 Tool

定义文件：

- `src/Tool.ts`

作用：

- 统一模型工具调用的声明与执行
- 统一权限、schema、并发与展示描述

关键结构：

- `Tool`
- `ToolUseContext`
- `ToolResult`
- `Tools`
- `buildTool()`

### 1.3 QueryParams

定义文件：

- `src/query.ts`

作用：

- 描述一次完整 `query()` 调用的输入

关键字段围绕：

- 消息
- system prompt / context
- toolUseContext
- fallback / budget / turns
- deps

### 1.4 AppState

定义文件：

- `src/state/AppStateStore.ts`

作用：

- 保存 REPL 层的完整应用状态
- 承接 UI、MCP、插件、任务、通知、权限、remote、bridge 等状态

### 1.5 Plugin / MCP 类型

当前另外两组重要类型入口是：

- `src/types/plugin.ts`
- `src/services/mcp/types.ts`

它们分别定义：

- plugin 可提供哪些组件
- MCP server 连接、配置与客户端状态如何表达

## 2. 结构化 I/O 与控制协议

文件：

- `src/cli/structuredIO.ts`
- `src/entrypoints/sdk/*`

这一层值得单独认识，因为它不只是“打印 JSON”，而是带有控制消息的协议边界。

当前代码里可确认的重要内容包括：

- `stream-json` / 结构化输出
- `control_request`
- `control_response`
- `control_cancel_request`
- 工具权限请求的转发与回收

因此 SDK / headless 模式和 REPL 模式共享的是核心执行面，不共享 UI 层协议。

- `src/services/oauth/client.ts`

## 4. 当前高频协议边界

从当前代码能确认的协议边界包括：

- Anthropic SDK message stream
- MCP JSON-RPC 与多传输接入
- WebSocket remote session stream
- direct-connect WebSocket
- structured I/O control protocol
- 终端 stdin / stdout 事件流
- Chrome native / browser 接入 fast-path

## 5. 当前高频 HTTP / WS 路径

下面这些是当前代码里高频出现、且足以建立认知模型的路径：

### 5.1 Session / remote / viewer

- `GET/POST {BASE_API_URL}/v1/sessions`
- `GET {BASE_API_URL}/v1/sessions/{sessionId}`
- `GET {BASE_API_URL}/v1/sessions/{sessionId}/events`
- `POST {BASE_API_URL}/v1/sessions/{sessionId}/archive`
- `WS  {BASE_API_URL}/v1/sessions/ws/{sessionId}/subscribe`
- `GET {BASE_API_URL}/v1/session_ingress/session/{sessionId}`

### 5.2 Files API

- `GET {baseUrl}/v1/files`
- `POST {baseUrl}/v1/files`
- `GET {baseUrl}/v1/files/{fileId}/content`

### 5.3 MCP / OAuth

- MCP server transport URLs 由配置决定
- OAuth / profile / usage 相关请求由 `services/oauth` 与 `utils/auth` 统一封装

## 6. 这一层的意义

接口层最重要的价值，不是穷举所有 API，而是明确三件事：

- 内部统一抽象是什么
- 外部真实协议是什么
- 两者之间由哪个适配层负责转换
