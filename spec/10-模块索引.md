# 10. 模块索引

## 1. 阅读原则

这个仓库模块很多，但第一次理解系统时，优先按问题域切分会更有效。

建议先回答：

1. 这是启动问题，还是运行中问题
2. 是本地 REPL，还是远程会话问题
3. 是命令层、工具层，还是协议层问题

## 2. 启动与模式分流

优先看：

- `src/entrypoints/*`
- `src/main.tsx`
- `src/cli/*`
- `src/server/*`

回答的问题：

- 进程怎么启动
- 为什么走到当前模式
- 当前是 REPL、非交互、MCP、remote 还是 bridge

## 3. REPL 与状态

优先看：

- `src/screens/REPL.tsx`
- `src/components/*`
- `src/ink/*`
- `src/state/*`
- `src/bootstrap/state.ts`
- `src/hooks/*`

回答的问题：

- 输入怎么进入系统
- UI 为什么会刷新
- 消息、权限、弹层、队列怎么协作

## 4. query 与 tools

优先看：

- `src/query.ts`
- `src/query/*`
- `src/Tool.ts`
- `src/tools.ts`
- `src/services/tools/*`
- `src/services/api/claude.ts`

回答的问题：

- agent turn 如何执行
- tool_use 如何被识别和执行
- 工具并发为何安全
- deferred tools / ToolSearch 如何生效

## 5. commands / skills / plugins

优先看：

- `src/commands.ts`
- `src/types/command.ts`
- `src/skills/*`
- `src/plugins/*`
- `src/utils/plugins/*`

回答的问题：

- slash command 如何定义
- skill 如何注入命令视图
- workflow command / dynamic skill 如何进入命令视图
- plugin 如何扩展命令、skills、hooks、MCP 等能力

## 5.1 task / agent / coordinator

优先看：

- `src/tools/AgentTool/*`
- `src/tasks/*`
- `src/coordinator/coordinatorMode.ts`

回答的问题：

- task / teammate / agent 如何表达
- 主线程与子代理如何分工
- coordinator mode 如何裁剪工具面

## 6. MCP

优先看：

- `src/services/mcp/*`
- `src/entrypoints/mcp.ts`
- `src/tools/MCPTool/*`
- `src/tools/McpAuthTool/*`

回答的问题：

- 外部 MCP server 如何接入
- Claude Code 如何作为 MCP server 暴露工具
- tools / prompts / resources 如何进入运行时

## 7. remote / bridge / direct connect / viewer

优先看：

- `src/remote/*`
- `src/bridge/*`
- `src/server/*`
- `src/hooks/useRemoteSession.ts`
- `src/hooks/useDirectConnect.ts`
- `src/hooks/useSSHSession.ts`
- `src/hooks/useAssistantHistory.ts`
- `src/assistant/sessionHistory.ts`

回答的问题：

- 远端消息如何回流本地
- bridge 如何把本地环境暴露给远端
- direct connect 如何接入 REPL
- `claude ssh` 如何把远端执行面接回本地 REPL
- assistant viewer 如何加载历史

## 8. API / OAuth / 外部系统

优先看：

- `src/services/api/*`
- `src/services/oauth/*`
- `src/cli/structuredIO.ts`
- `src/utils/auth.ts`

回答的问题：

- 外部 HTTP / WS / OAuth 协议在哪里封装
- 结构化 I/O / SDK 控制协议在哪里处理
- provider 切换和认证状态在哪里落地

## 9. 当前工作树异常域

如果你在代码里看到这些路径，不要默认它们在当前工作树可读：

- `connectHeadless`
- `createSSHSession`
- `sessionDiscovery`

它们目前属于“源码仍有引用，但实现缺失”的异常项。
