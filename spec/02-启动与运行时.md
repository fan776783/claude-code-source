# 02. 启动与运行时

## 1. 启动层分工

### 1.1 最外层入口

`src/entrypoints/cli.tsx` 是最外层 bootstrap 入口。

它的职责是：

- 处理极轻量 fast-path
- 延迟导入重模块
- 在真正进入 `main.tsx` 前完成模式分流

当前代码里可确认的 fast-path 包括：

- `--version`
- `--dump-system-prompt`
- `--claude-in-chrome-mcp`
- `--chrome-native-host`
- `--computer-use-mcp`
- `--daemon-worker`
- `remote-control` / `rc` / `remote` / `sync` / `bridge`
- `daemon`
- `ps` / `logs` / `attach` / `kill`
- template jobs
- `environment-runner`
- `self-hosted-runner`
- `--worktree` + `--tmux`

### 1.2 主启动器

`src/main.tsx` 是真正的主程序入口和装配中心。

它负责：

- early argv / env 二次分流
- 初始化配置、认证、遥测、清理逻辑
- 解析 CLI 参数
- 构建 commands、tools、MCP、plugins、AppState
- 决定运行模式并进入对应路径

当前代码里可确认的 early 处理包括：

- `mcp serve` 相关入口判定
- `cc://` / `cc+unix://` URL 重写
- `claude assistant [sessionId]` viewer 模式判定
- `claude ssh <host> [dir]` 远程模式判定
- protocol handler / deep link 早退路径

### 1.3 初始化层

`src/entrypoints/init.ts` 负责早期初始化。

主要职责：

- 启用配置系统
- 注入环境变量
- TLS / 证书准备
- telemetry / analytics 初始化
- 清理回调注册
- 非交互场景下的配置错误处理

## 2. 主运行模式

从当前工作树看，`main.tsx` 最终会把会话导向这些主模式：

- 交互式 REPL
- 非交互 print / structured I/O / SDK
- MCP Server
- remote session
- bridge / remote-control
- direct connect
- assistant viewer

需要注意：

- 这些是“当前能从代码确认的主模式”
- 不是所有 fast-path 的穷举列表

## 3. 非交互运行时

这条线的关键入口主要是：

- `src/cli/print.ts`
- `src/cli/structuredIO.ts`
- `src/cli/transports/*`
- `src/QueryEngine.ts`

它们分别承担：

- REPL 之外的 headless 执行
- 结构化输入输出协议
- SSE / WebSocket / hybrid 传输
- 查询生命周期与会话状态管理

## 4. 运行时装配中心

把 `main.tsx` 理解成“装配器”更准确。

它会统一装配：

- settings / config / auth
- bootstrap state
- AppState store
- built-in commands
- built-in tools
- MCP clients / commands / resources
- plugin 与 skill 结果
- 本地 REPL 或远程会话变体

## 6. 当前工作树异常

下面这些能力**在源码中仍被引用**，但对应实现文件在当前工作树缺失：

- `connectHeadless`
- `createSSHSession`
- `sessionDiscovery`

这意味着：

- 文档不能再把它们当作“当前已存在的实现入口”
- 但阅读 `main.tsx` 时仍会看到它们的动态导入引用

如果你正在排查这几条线，需要先确认这是裁剪仓库、未提交文件，还是重构中的中间态。

## 7. 启动数据流

```text
命令行参数
  -> entrypoints/cli.tsx fast-path 分流
  -> main.tsx early argv / env 判定
  -> init.ts 初始化配置 / 认证 / 遥测
  -> 装配 AppState / Tools / Commands / MCP / Plugins
  -> 进入 REPL / print / MCP / remote / bridge / direct connect / viewer
```
