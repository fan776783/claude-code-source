# 13. 关键目录树

## 1. 使用方式

这份目录树不是完整文件树，而是“当前最值得先看”的目录导航。

适合场景：

- 第一次进仓库，不知道从哪里下手
- 知道问题属于哪一层，但不知道目录在哪
- 想先建立 `src/` 的空间感

## 2. 当前高价值目录树

```text
src/
├── entrypoints/                启动入口与初始化
│   ├── cli.tsx
│   ├── init.ts
│   └── mcp.ts
│
├── cli/                        非交互 / structured I/O / transport
├── screens/                    主屏幕
│   └── REPL.tsx
├── components/                 交互组件
├── ink/                        终端 UI 运行时
├── hooks/                      REPL 与远程相关 hooks
│
├── state/                      UI 应用状态
│   ├── AppStateStore.ts
│   ├── AppState.tsx
│   ├── store.ts
│   └── onChangeAppState.ts
│
├── bootstrap/                  进程级全局状态
│   └── state.ts
│
├── query/                      query 辅助逻辑
│   ├── deps.ts
│   └── stopHooks.ts
│
├── services/
│   ├── api/                    模型与 HTTP API 适配
│   ├── mcp/                    MCP client / auth / config
│   ├── oauth/                  OAuth 与账户
│   ├── compact/                上下文压缩
│   ├── tools/                  工具执行编排
│   ├── analytics/              telemetry / growthbook / logging
```

## 3. 其他当前存在的能力域

除了上面的主链路目录，当前工作树还存在这些辅助能力域：

- `buddy/`
- `memdir/`
- `moreright/`
- `outputStyles/`
- `voice/`

它们不是第一次读代码最该优先看的目录，但确实属于当前仓库。

## 4. 按问题类型找目录

### 4.1 启动问题

优先看：

- `src/entrypoints/`
- `src/main.tsx`
- `src/cli/`
- `src/server/`

### 4.2 输入、消息、UI 问题

优先看：

- `src/screens/`
- `src/components/`
- `src/ink/`
- `src/hooks/`
- `src/state/`

### 4.3 模型、query、工具问题

优先看：

- `src/query.ts`
- `src/query/`
- `src/services/api/`
- `src/services/tools/`
- `src/tools/`

### 4.4 命令、skill、plugin 扩展问题

优先看：

- `src/commands/`
- `src/skills/`
- `src/plugins/`
- `src/utils/plugins/`

### 4.5 MCP、远程、桥接问题

优先看：

- `src/services/mcp/`
- `src/remote/`
- `src/bridge/`
- `src/server/`
- `src/assistant/`

## 5. 当前工作树异常

不要把下面这些目录/文件当成当前已存在的目录树节点：

- `ssh`
- `connectHeadless`
- `sessionDiscovery`

它们在源码引用中出现，但不在当前工作树中。
