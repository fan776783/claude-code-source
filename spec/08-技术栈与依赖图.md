## 3. 关键依赖关系说明

### 3.1 `main.tsx` 是装配中心

它依赖：

- config / auth / bootstrap state
- commands 聚合
- tools 聚合
- MCP 与 plugin 结果
- 交互式与非交互式入口

所以它更像装配器，不是单纯业务模块。

### 3.2 `REPL.tsx` 是交互协调中心

它依赖：

- AppState
- PromptInput
- `query()`
- 各类远程 hook
- permission / dialog / notification 体系

所以它本质上是会话协调器，而不是单纯视图组件。

### 3.3 `query.ts` 是代理内核

它依赖：

- message 正规化
- 模型调用
- 工具执行
- compaction / hooks / retry / budget

### 3.4 `services/*` 是外部系统适配层

这里的服务层主要职责不是 UI 服务，而是：

- 统一外部协议
- 封装认证
- 转换成内部统一消息 / 工具 / 资源结构

## 4. 架构特征

### 4.1 优点

- 分层仍然清晰：入口、UI、query、tools、services、state 各有主责
- 扩展面丰富：commands、skills、plugins、MCP 四层扩展同时存在
- 运行形态多：REPL、print、SDK、MCP、remote、bridge、viewer 并存
- 工具系统完整：权限、并发、deferred tools、tool search 都已成型

### 4.2 复杂度来源

- `main.tsx` 与 `REPL.tsx` 体量大、协调职责重
- feature flags 很多
- 运行模式与传输协议较多
- 当前工作树还存在“源码引用但实现缺失”的异常项

## 5. 当前版本结论

从当前工作树看，这个项目最适合被理解为：

**一个面向终端与远程协作场景的代理式 CLI 运行平台。**
