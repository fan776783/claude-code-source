# 09. 关键时序图

## 1. CLI 启动时序

```text
用户执行 claude
  -> entrypoints/cli.tsx
  -> main.tsx
  -> early argv / env 分流
  -> init.ts
  -> 解析 settings / auth / telemetry
  -> 装配 commands / tools / MCP / plugins / AppState
  -> 选择运行模式
      -> REPL
      -> 非交互 print / SDK
      -> MCP server
      -> remote / bridge
      -> direct connect
      -> assistant viewer
```

## 2. Prompt 提交主时序

```text
用户输入
  -> ink/parse-keypress.ts
  -> PromptInput.tsx
  -> screens/REPL.tsx:onSubmit
  -> handlePromptSubmit.ts
  -> processUserInput()
  -> onQuery()
  -> query()
  -> setMessages / setAppState
  -> UI 重绘
```

补充：

- `local-jsx` / immediate 命令会在 `onSubmit` 内提前分流
- remote / bridge / direct connect 的输入不会全部走本地 query 主链

## 3. query 主循环时序

```text
REPL.onQuery
  -> query.ts
  -> 构造 systemPrompt / userContext / systemContext
  -> deps.callModel()
  -> services/api/claude.ts
  -> 流式接收 assistant 输出
  -> 主动收集 tool_use
  -> 若出现 tool_use:
       -> toolOrchestration / StreamingToolExecutor
       -> 收集 tool_result
       -> 回灌 query 下一轮
  -> drain queued commands / task-notification attachments
  -> 注入 memory / skill prefetch attachments
  -> 若需要 compact / hooks / retry:
       -> 执行对应分支
  -> turn 完成
```

## 4. 工具执行时序

```text
assistant 返回 tool_use
  -> services/tools/toolOrchestration.ts
  -> 判断 isConcurrencySafe
      -> 并发安全片段并发执行
      -> 其他批次串行执行
  -> Tool.call(args, context, canUseTool, parentMessage)
  -> ToolResult / newMessages / contextModifier
  -> query.ts
  -> 作为 tool_result 再次送给模型
```

## 5. MCP Client 时序

```text
启动或配置变更
  -> services/mcp/config.ts
  -> services/mcp/client.ts
  -> 建立连接
  -> 拉取 tools / prompts / resources
  -> prompts -> commands
  -> skill resources -> skill commands
  -> AppState.mcp 更新
  -> 合并到工具池 / 命令池 / 资源池
```

## 6. MCP Server 时序

```text
外部 MCP Client
  -> entrypoints/mcp.ts
  -> ListTools
  -> 返回内建工具定义
  -> CallTool(name, args)
  -> 复用内部 Tool.call()
  -> 返回 MCP 结构化结果
```

## 7. Remote session 时序

```text
本地 REPL 提交输入
  -> remote-safe command 过滤
  -> activeRemote.sendMessage()
  -> HTTP POST 到 remote session
  -> WebSocket 订阅远端事件
  -> 远端执行
  -> SDKMessage 回流
  -> sdkMessageAdapter
  -> useRemoteSession
  -> setMessages / setAppState
```

## 8. Bridge 时序

```text
bridge 初始化
  -> bridge/initReplBridge.ts
  -> 建立 bridge 会话
  -> 接收 inbound message / control
  -> 注入本地执行面
  -> 本地结果写回 bridge session
  -> 远端看到同步状态
```

## 9. Assistant viewer 历史时序

```text
viewer 模式进入 REPL
  -> useAssistantHistory()
  -> assistant/sessionHistory.ts
  -> 拉取 /v1/sessions/{id}/events
  -> 分页转换为本地消息
  -> prepend 到消息列表
  -> REPL 渲染历史 + 实时事件
```
