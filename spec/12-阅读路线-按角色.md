# 12. 阅读路线按角色

## 1. 新同学快速上手

目标：

- 建立整体认知
- 知道系统主要分层
- 能快速定位主入口和主循环

推荐顺序：

1. `spec/README.md`
2. `spec/01-项目概览.md`
3. `spec/02-启动与运行时.md`
4. `spec/03-REPL与状态管理.md`
5. `spec/04-查询执行与工具系统.md`
6. `spec/09-关键时序图.md`

建议跟读源码：

- `src/entrypoints/cli.tsx`
- `src/main.tsx`
- `src/screens/REPL.tsx`
- `src/query.ts`
- `src/Tool.ts`

## 3. 问题排查

目标：

- 快速找到问题发生在哪一层
- 缩小到具体模块和文件

推荐顺序：

1. `spec/09-关键时序图.md`
2. `spec/10-模块索引.md`
3. `spec/11-源码文件映射表.md`
4. 对应主题文档：
   - 启动问题看 `02`
   - 交互问题看 `03`
   - 模型 / 工具问题看 `04`
   - MCP / 远程问题看 `06`

常见排查入口：

- 启动异常
  看 `src/entrypoints/cli.tsx`、`src/main.tsx`、`src/entrypoints/init.ts`

- 输入或 UI 异常
  看 `src/screens/REPL.tsx`、`src/components/PromptInput/PromptInput.tsx`、`src/ink/parse-keypress.ts`

- 模型响应异常
  看 `src/query.ts`、`src/services/api/claude.ts`、`src/services/api/errors.ts`

- SDK / headless 异常
  看 `src/QueryEngine.ts`、`src/cli/print.ts`、`src/entrypoints/sdk/*`

- direct connect / remote / viewer 异常
  看 `src/server/*`、`src/remote/*`、`src/bridge/*`、`src/hooks/useAssistantHistory.ts`

- ssh 远程异常
  看 `src/hooks/useSSHSession.ts`、`src/main.tsx`

- 工具执行异常
  看 `src/services/tools/toolOrchestration.ts`、`src/services/tools/StreamingToolExecutor.ts`、`src/tools/*`

- MCP 异常
  看 `src/services/mcp/*`

## 4. 功能扩展

目标：

- 知道应该往哪一层扩展
- 避免把能力加错位置

如果要新增：

- 新工具
  先看 `spec/04-查询执行与工具系统.md`
  然后读 `src/Tool.ts`、`src/tools.ts`、对应 `src/tools/*`

- 新 slash command
  先看 `spec/05-命令-Skills-插件.md`
  然后读 `src/types/command.ts`、`src/commands.ts`、对应 `src/commands/*`

- 新 Skill
  先看 `spec/05-命令-Skills-插件.md`
  然后读 `src/skills/loadSkillsDir.ts`、`src/tools/SkillTool/SkillTool.ts`

- 新 plugin 能力
  先看 `spec/05-命令-Skills-插件.md`
  然后读 `src/plugins/builtinPlugins.ts`、`src/utils/plugins/pluginLoader.ts`

- 新 MCP 接入
  先看 `spec/06-MCP与远程桥接.md`
  然后读 `src/services/mcp/client.ts`、`src/services/mcp/config.ts`、`src/services/mcp/auth.ts`

- 新远程能力
  先看 `spec/06-MCP与远程桥接.md`
  然后读 `src/remote/*`、`src/bridge/*`、`src/server/*`、`src/hooks/useSSHSession.ts`、`src/assistant/sessionHistory.ts`
