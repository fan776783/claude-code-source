# 01. 项目概览

## 1. 项目定位

这个仓库对应 Anthropic 的终端代理工具 `Claude Code`。

从当前代码树看，它不是单一聊天 CLI，而是同时具备这些角色：

- 终端交互式编码助手
- 非交互 / 结构化输出执行器
- MCP Client / MCP Server
- 远程会话与 bridge 控制端
- commands、skills、plugins、tools 的统一扩展平台

## 2. 发布形态

仓库根目录同时存在：

- `src/`
- `cli.js`
- `cli.js.map`
- `package.json`
- `bun.lock`

这说明当前发布策略是：

- 源码在 `src/`
- 发布产物是 bundle 后的单文件 CLI
- `package.json` 依赖表并不能完整反映真实运行时组成

因此读架构时，应以 `src/` 内部模块为准。

## 3. 当前代码树的高价值目录

下面这组是**当前工作树里真实存在**、且对理解系统最重要的目录域：

```text
src/
├── entrypoints/        CLI / MCP / SDK 启动入口
├── cli/                非交互、structured I/O、传输层
├── screens/            REPL 等主屏幕
├── components/         交互 UI 组件
├── ink/                终端 UI 运行时与输入解析
├── state/              AppState store 与副作用同步
├── bootstrap/          进程级全局状态
├── query/              query 辅助逻辑
├── tools/              模型可调用工具
├── commands/           slash commands
├── skills/             本地与 bundled skills
├── plugins/            built-in plugin 注册
├── services/           API、MCP、OAuth、analytics、compact 等服务层
├── remote/             remote session 适配
├── bridge/             remote control / bridge 运行时
├── server/             direct connect 会话能力
├── assistant/          assistant viewer 相关能力
├── tasks/              task / teammate / agent 运行时
├── coordinator/        coordinator mode 协调逻辑
├── hooks/              React hooks 与会话辅助逻辑
├── types/              核心类型定义
├── keybindings/        键位体系
└── utils/              通用基础设施与胶水层
```

另外还存在一些更偏能力域的目录，例如：

- `buddy/`
- `memdir/`
- `moreright/`
- `outputStyles/`
- `voice/`

它们也是当前工作树的一部分，但不是第一次读代码时最该先看的入口。

## 5. 三个核心抽象

如果只保留三件事来理解这个仓库，优先看：

1. `AppState`
   交互式运行时的统一状态源。

2. `Tool` / `ToolUseContext`
   模型可调用能力、权限与上下文注入的统一契约。

3. `query()`
   把消息、模型、工具、压缩、重试和停止钩子串起来的代理循环。

## 6. 当前工作树异常

当前仓库里有一类需要单独认识的现象：

- 部分源码仍引用了缺失文件
- 这些缺失项不能被视为“当前可读实现”

目前已能确认的重要异常包括：

- `src/main.tsx` 动态导入 `./server/connectHeadless.js`
- `src/main.tsx` 动态导入 `./ssh/createSSHSession.js`
- `src/main.tsx` 动态导入 `./assistant/sessionDiscovery.js`

因此后续文档会把这几条线区分成：

- 当前工作树可确认的实现
- 当前源码仍有引用、但实现文件缺失的异常项
