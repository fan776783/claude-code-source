# Claude Code 项目规格索引

## 1. 说明

本目录基于**当前检出的工作树**整理 Claude Code 的结构说明。

文档目标有两点：

- 帮你快速理解系统主链路
- 帮你把问题定位到真实存在的源码入口

如果某个能力在代码里还有引用、但对应文件在当前工作树缺失，本文档会显式标注为“仓库现状异常”，而不会继续把它当成正常入口。

## 2. 推荐阅读顺序

### 2.1 10 分钟速读

1. `01-项目概览.md`
2. `02-启动与运行时.md`
3. `03-REPL与状态管理.md`
4. `04-查询执行与工具系统.md`
5. `09-关键时序图.md`

配套源码最短路径：

1. `src/entrypoints/cli.tsx`
2. `src/main.tsx`
3. `src/screens/REPL.tsx`
4. `src/query.ts`
5. `src/services/api/claude.ts`
6. `src/Tool.ts`

## 3. 每篇文档解决什么问题

- `01-项目概览.md`
  项目是什么，当前代码树大致怎么分层。

- `02-启动与运行时.md`
  程序怎么从 CLI 进入 REPL、非交互、MCP、remote、bridge 等模式。

- `03-REPL与状态管理.md`
  输入如何进入系统，REPL 如何协调状态、消息、权限和远程会话。

- `04-查询执行与工具系统.md`
  `query()` 如何运行，工具如何注册、延迟加载、并发编排。

- `05-命令-Skills-插件.md`
  slash command、skill、plugin 如何统一进入命令视图与执行面。

- `06-MCP与远程桥接.md`
  MCP client/server、remote、bridge、direct connect、assistant viewer 的边界。

- `07-接口与协议.md`
  核心内部契约、结构化 I/O、外部 API 与协议边界。

- `08-技术栈与依赖图.md`
  当前可确认的技术栈、依赖关系与复杂度来源。

- `09-关键时序图.md`
  以流程图方式串起主链路。

- `10-模块索引.md`
  先按问题域找模块，再落到文件。

- `11-源码文件映射表.md`
  当前最值得直接打开的关键文件。

- `12-阅读路线-按角色.md`
  按不同读者目标给出最短阅读路径。

- `13-关键目录树.md`
  建立 `src/` 的空间感，只列当前高价值目录。

- `14-关键源码链接索引.md`
  按主题聚合源码入口，适合边读边跳代码。

- `15-常见问题定位手册.md`
  按故障类型给出最快的文档入口与源码入口。

## 4. 当前版本结论

从当前工作树看，Claude Code 最适合被理解为：

**一个支持终端交互、结构化 I/O、MCP 扩展、插件扩展、远程会话和多模型后端的代理式 CLI 平台。**
